## 角色类型

系统角色分为三类：

1. 超级管理员：`super_admin`
2. 店长：`manager`
3. 普通员工：`employee`

职位和系统角色分开存储。职位用于描述员工岗位，系统角色用于控制权限。

## 超级管理员

超级管理员拥有全部功能：

- 查看所有门店。
- 新增、选择、删除门店。
- 查看任意门店下的员工。
- 给指定门店办理员工入职。
- 编辑员工信息。
- 办理员工离职。
- 删除已离职员工信息。
- 查看所有门店员工考勤。
- 管理所有门店排班。
- 使用设置页内的全局维护能力。

默认体验账号：

- 手机号：`15500000000`
- 角色：超级管理员

## 店长

店长只管理自己所属门店：

- 只能查看自己门店信息。
- 只能查看和管理本门店员工。
- 只能给本门店办理员工入职。
- 可以编辑本门店员工信息。
- 可以办理本门店员工离职。
- 员工离职后，可以删除本门店员工信息。
- 只能查看本门店员工考勤。
- 只能管理本门店排班。
- 不能新增或删除门店。
- 不能查看其他门店员工、考勤和排班。

默认体验账号：

- 手机号：`15522013798`
- 角色：店长

## 普通员工

普通员工包含收银和网管：

- 进入员工端。
- 可以上下班打卡。
- 可以查看本门店排班。
- 可以查看自己的排班信息。
- 不能管理员工。
- 不能新增、删除或切换门店。
- 不能调整排班。
- 不能查看其他门店信息。

默认体验账号：

- 手机号：`15922251233`
- 角色：普通员工

## 数据字段

员工角色关系表使用以下核心字段：

- `staffId`：员工 ID。
- `phone`：手机号，用于登录识别。
- `storeId`：所属门店 ID。超级管理员可为空或配置多个门店在员工表 `storeIds` 中。
- `position`：职位枚举，例如 `manager`、`cashier`、`network`、`staff`。
- `positionText`：职位中文名。
- `role`：系统角色，取值为 `super_admin`、`manager`、`employee`。

员工表使用 `storeIds` 表示员工所属门店范围。店长和普通员工通常只属于一个门店，超级管理员可以属于多个门店或拥有全局权限。

## 登录判定

用户进入小程序后，通过手机号查询员工角色关系：

1. `role = super_admin`：进入超级管理员工作台。
2. `role = manager`：进入店长工作台。
3. `role = employee`：进入员工端。
4. 旧数据中 `role = admin` 时，按 `manager` 店长处理。
5. 如果没有角色关系，则不允许登录业务端。

## 权限原则

所有管理端页面都必须按角色过滤数据：

- 超级管理员：不过滤门店范围。
- 店长：按当前登录员工的 `storeIds` 过滤。
- 普通员工：按当前登录员工和所属门店过滤。

页面不能只隐藏入口，还必须在数据读取和保存时应用同样的权限范围。
